# Avances Modulo Logistica - 04/05/2026

## Resumen Ejecutivo

El modulo de Logistica ya quedo operativo localmente con PostgreSQL y con integracion funcional contra el modulo Restaurantes desplegado en Fly.io.

Se comprobo el flujo end-to-end:

```txt
Restaurantes crea pedido real
Logistica consume payload logistico del pedido
Logistica crea entrega
Repartidor simulado acepta
Logistica notifica a Restaurantes: ACCEPTED
Restaurantes cambia a Listo
Repartidor marca recogida
Logistica notifica a Restaurantes: PICKED_UP
Restaurantes cambia a En Camino
Repartidor marca entregado
Logistica notifica a Restaurantes: DELIVERED
Restaurantes cambia a Entregado
```

## Integracion Con Restaurantes

### Base URL

```txt
https://restaurantes.fly.dev/api
```

Configurable en:

```txt
RESTAURANTES_API_URL
```

Default en `src/config/env.js`:

```txt
https://restaurantes.fly.dev/api
```

### Endpoint Consumido Por Logistica

```http
GET /api/restaurantes/:restaurante_id/pedidos/:pedido_id/logistica
```

Este endpoint ya funciona y devuelve payload logistico.

### Endpoint Usado Para Notificar Estado

```http
PUT /api/restaurantes/:restaurante_id/pedidos/:pedido_id/estado
```

Logistica envia:

```json
{
  "estado_id": 5,
  "motivo": "PICKED_UP",
  "metadata": {
    "entrega_id": 4,
    "estado_logistica": "en_ruta",
    "repartidor_id": 101,
    "comentario": "Pedido recogido",
    "timestamp": "2026-05-04T19:44:04.742Z"
  }
}
```

### Mapeo De Estados Confirmado

| Evento Logistica | Estado Logistica | Estado Restaurante |
|---|---|---|
| `ACCEPTED` | `asignada` | `4 - Listo` |
| `PICKED_UP` | `en_ruta` | `5 - En Camino` |
| `DELIVERED` | `entregada` | `6 - Entregado` |
| `CANCELLED` | `cancelada` | `7 - Cancelado` |
| `FAILED` | `fallida` | `7 - Cancelado` |
