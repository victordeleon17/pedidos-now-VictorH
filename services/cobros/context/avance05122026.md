# Avance 05/12/2026

## Resumen

Se completo el despliegue del servicio `cobros` en Fly.io y se migro la configuracion de base de datos desde MySQL hacia PostgreSQL usando Neon.

## Acciones realizadas

- Se intento inicialmente desplegar MySQL dentro de Fly.io como app separada.
- La app MySQL fallo por problemas de inicializacion/volumen, por lo que se elimino completamente.
- Se eliminaron los recursos de MySQL en Fly.io:
  - App `cobros-mysql`.
  - Maquina asociada.
  - Volumen persistente `mysql_data`.
- Se adapto el proyecto para PostgreSQL:
  - Se reemplazo `mysql2` por `pg`.
  - Sequelize cambio de dialecto `mysql` a `postgres`.
  - Se actualizo `DB_PORT` de `3306` a `5432`.
  - Se actualizo `DB_NAME` a `cobrosdb`.
  - Se habilito SSL para conexion con Neon.
- Se adapto la sintaxis de migraciones y seeders:
  - UUID ahora usa `Sequelize.UUIDV4`.
  - Se elimino uso de `INTEGER.UNSIGNED`.
  - Seeders cambiaron de `ON DUPLICATE KEY UPDATE` a `ON CONFLICT ... DO UPDATE`.
- Se adapto `src/db.js` para usar `pg` en lugar de `mysql2/promise`.
- Se creo el proyecto Neon `cobrosdb`.
- Se creo la base de datos `cobrosdb` en Neon.
- Se configuraron secrets de Fly.io para conectar `cobros-api` con Neon.
- Se creo `fly.toml` para el servicio `cobros-api`.
- Se redesplego la API en Fly.io.

## Validaciones realizadas
